# 橘猫智酷商城小程序 部署与运维手册

**版本**：1.0

---

## 1. 环境要求

| 项目 | 要求                                                    |
|------|-------------------------------------------------------|
| 服务器 | Linux 推荐（CentOS 7+ / Ubuntu 18+）或 Windows（如 phpstudy） |
| PHP | 8.0+，扩展：mbstring、json、openssl、pdo_mysql、curl、fileinfo |
| MySQL | 5.7+ 或 MariaDB 10.2+                                  |
| Web 服务器 | Nginx 1.18+ 或 Apache 2.4+（需 mod_rewrite）              |
| 小程序前端 | 微信开发者工具 + UniApp 编译                                   |

## 2. 后端部署（ThinkPHP）

### 2.1 代码放置

- 将 `php/` 目录放到 Web 可访问目录，**文档根目录必须指向 `php/public`**，不能指向 `php`，避免暴露源码。

### 2.2 依赖安装

```bash
composer install --no-dev
```

若未安装 Composer：`curl -sS https://getcomposer.org/installer | php`，再 `php composer.phar install --no-dev`。

### 2.3 目录权限

- `runtime/`、`public/uploads/` 需可写：
  ```bash
  chmod -R 755 runtime public/uploads
  chown -R www-data:www-data runtime public/uploads   # 用户按实际运行身份调整
  ```

### 2.4 环境变量

- 在 `php/` 下复制 `.env.example` 为 `.env`（若无则新建），配置：
  - 数据库：`DB_DRIVER`、`DB_HOST`、`DB_NAME`、`DB_USER`、`DB_PASS`、`DB_PORT`、`DB_CHARSET`、`DB_PREFIX`
  - 应用：`APP_DEBUG`（生产建议 0）、`APP_TRACE` 等  
  详见《05-环境与配置说明.md》。

### 2.5 数据库初始化

```bash
mysql -u root -p <数据库名> < php/database/schema_optimized.sql
```

- 若表已存在，仅需新增字段（如 mp_server.corpId），请自行执行 ALTER 或合并脚本。

### 2.6 Nginx 配置示例

```nginx
server {
    listen 80;
    server_name mp.goomooplay.com;   # 替换为实际域名
    root /var/www/public;
    index index.php;

    location / {
        if (!-e $request_filename) {
            rewrite ^(.*)$ /index.php?s=$1 last;
            break;
        }
    }

    location ~ \.php($|/) {
        fastcgi_pass unix:/run/php/php7.4-fpm.sock;   # 或 127.0.0.1:9000
        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
        include fastcgi_params;
        fastcgi_param PATH_INFO $fastcgi_path_info;
    }

    location ~* \.(jpg|jpeg|png|gif|ico|css|js|woff2)$ {
        expires 7d;
        add_header Cache-Control "public, immutable";
    }
}
```

- HTTPS：使用 Let’s Encrypt 或已有证书，在 server 内增加 `listen 443 ssl` 及 `ssl_certificate`、`ssl_certificate_key`。

### 2.7 Apache 配置

- 启用 `mod_rewrite`，保证 `php/public/.htaccess` 生效（若为空，可配置一条 FallbackResource 到 index.php，或使用项目自带 router.php 重写规则）。

## 3. 小程序前端部署

### 3.1 安装依赖与编译

```bash
cd uniapp
npm install
# 微信小程序
npm run dev:mp-weixin
# 或
npm run build:mp-weixin
```

- 编译产物在 `uniapp/unpackage/dist/dev/mp-weixin` 或 `dist/build/mp-weixin`。
- 使用微信开发者工具打开该目录，上传为体验版/正式版。

### 3.2 配置后端地址

- 在 `uniapp/utils/request.js` 中修改 `baseURL`：
  - 生产：`https://mp.goomooplay.com/api`
  - 开发：可改为 `http://你的本机或内网IP/api`（微信小程序需在小程序后台配置 request 合法域名或开发时勾选“不校验合法域名”）。

### 3.3 小程序后台配置

-  request 合法域名：`https://mp.goomooplay.com`
- 若用微信支付，需配置支付商户号、支付目录等。
- 若用企业微信客服，需配置客服相关参数。

## 4. 支付与回调

### 4.1 汇付/微信支付

- 在支付扩展或配置中填写商户号、密钥、回调地址（如 `https://mp.goomooplay.com/api/order/notify`）。
- 确保回调 URL 可从公网访问，且为 HTTPS。

### 4.2 支付回调排查

- 查看 `php/runtime/log` 及缓存键（如 notify 相关）排查是否收到回调、是否执行成功。
- 订单未更新时检查：订单状态是否允许支付、金额是否一致、是否重复通知导致幂等处理。

## 5. 日常运维

### 5.1 日志

- 应用日志：`php/runtime/log/` 按日期与模块分文件。
- Nginx/Apache 访问与错误日志：见各自配置。

### 5.2 缓存

- 默认文件缓存在 `php/runtime/cache/`。
- 清理缓存：可删 `runtime/cache` 下子目录或使用 ThinkPHP 命令（若有配置）：`php think clear --cache`。

### 5.3 备份

- 定期备份 MySQL（含 mp_ 表）。
- 备份 `public/uploads` 上传文件。
- `.env` 与密钥类配置单独备份，勿提交版本库。

### 5.4 更新发布

1. 备份数据库与 uploads。
2. 拉取代码，执行 `composer install --no-dev`（若有依赖变更）。
3. 执行数据库变更（若有 schema 升级脚本）。
4. 清理 runtime 缓存（可选）。
5. 小程序端重新编译并上传新版本。

### 5.5 监控建议

- 监控站点可用性（如 200）、订单/支付成功率。
- 监控磁盘空间（runtime、uploads、日志）。
- 生产环境关闭 `APP_DEBUG`，避免暴露错误详情。

## 6. 常见问题

| 现象 | 可能原因 | 处理 |
|------|----------|------|
| 500 错误 | 权限、语法、缺少扩展 | 看 runtime/log，检查 PHP 扩展与目录权限 |
| 接口 404 | 未重写或路由错误 | 确认 Nginx/Apache 重写指向 index.php，且访问路径为 /api/控制器/方法 |
| 登录/支付失败 | 域名未备案、未配置合法域名、回调不可达 | 配置小程序合法域名、保证回调 URL 公网 HTTPS 可访问 |
| 订单未更新 | 支付回调未执行或失败 | 查日志与缓存中回调记录，核对商户配置与回调地址 |
