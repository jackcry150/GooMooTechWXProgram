# 橘猫智酷商城小程序 系统架构与设计文档

**版本**：1.0

---

## 1. 系统概述

本系统为**橘猫智酷商城**品牌微信小程序电商与内容展示一体化项目，包含：

- **后端**：ThinkPHP8 多应用（API + 管理后台 adm）
- **前端**：UniApp（Vue）编译为微信小程序
- **主要能力**：用户登录、商品/贩售、购物车、订单、支付（汇付/微信）、地址、收藏、资讯、客服、积分（蜗壳）等

---

## 2. 技术栈

| 层级 | 技术             | 说明 |
|------|----------------|------|
| 后端框架 | ThinkPHP 8.x   | 多应用：api、adm |
| 语言 | PHP 8.0+       | - |
| 数据库 | MySQL 5.7+     | 表前缀 mp_，utf8mb4 |
| 缓存 | File           | config/cache.php，可扩展 Redis |
| 前端框架 | UniApp (Vue 2) | 编译目标：微信小程序、H5 |

| 第三方 | 管家婆、企业微信客服     | 订单同步、在线客服 |

## 5. 模块设计

### 5.1 用户与认证

- **小程序登录**：wx.login 取 code，后端调微信 jscode2session 得 openid，生成 token（加密 JSON：userId、openId、exp）。
- **请求鉴权**：小程序请求头带 `Authorization: {token}`，后端解密校验 userId。
- **用户表**：mp_user（openId、nickName、avatar、phone、snailShells、collectionCards 等）。

### 5.2 商品与贩售

- **商品**：mp_product，支持现货(type=1)/预售(type=2)、特别贩售(isSpecialSale)、分账字段。
- **贩售**：mp_product_sell 独立活动表；接口 /api/sell/latest 取 isSpecialSale=1 的最新商品，用于贩售页。
- **列表/详情**：/api/product/list、/api/product/detail。

### 5.3 购物车与订单

- **购物车**：mp_cart，按用户、商品、版本维度；支持数量修改、校验库存与限购。
- **订单**：mp_order，创建时仅存商品最小信息（productId、version、quantity）；支付成功后落库完整快照、扣减库存，并可选同步管家婆。
- **订单状态**：1 待支付、2 待发货、4 已取消、5 删除、6 待收货、7 已完成、8 已预定、9 退款成功、10 已付定金待付尾款。
- **退款**：refundStatus 与 status 配合；后台同意退款后 status 置为 9。

### 5.4 支付与积分

- **支付**：汇付/斗拱生成预支付参数，小程序 wx.requestPayment 调起微信支付；支付结果异步通知 /api/order/notify（或 Payment 控制器），成功后更新订单、扣库存、同步管家婆。
- **蜗壳**：确认收货时按订单实付金额 1 元=1 蜗壳，增加 mp_user.snailShells。

### 5.5 资讯与设置

- **资讯**：mp_news，固定三条 code：about（公司简介）、after_sale（售后说明）、service_agreement（服务协议）；仅后台编辑，无增删。
- **设置**：mp_setting 单条，站点名、链接、联系我们、地址、邮箱、客服链接等；前端 /api/setting/info 拉取。

### 5.6 客服与广告

- **客服**：mp_server，type=1 微信社区群、2 社交账号、3 微信客服（在线客服）；前端 /api/server/list?type=3 拉取在线客服，wx.openCustomerServiceChat 调起。
- **广告**：mp_ad，type=1 banner、2 会员页；/api/banner/list。

## 6. 安全与约定

- 所有需登录接口从 Header `Authorization` 取 token，校验失败返回 401。
- 订单、地址、购物车等写操作均校验 userId，防止越权。
- 支付回调需校验订单号、金额、状态，防篡改与重复处理。
- 敏感配置（数据库、微信 appid/secret、汇付参数）放在 .env，不纳入版本库。

---

## 7. 扩展与依赖

- **管家婆**：订单同步逻辑在 Order 控制器与 poguanjia 相关类中，需配置 token 等。
- **汇付/斗拱**：extend 或 vendor 内支付封装，配置商户号、回调地址等。
- **图库**：mp_gallery_category + mp_gallery，后台选择图片时使用。
