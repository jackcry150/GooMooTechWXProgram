# 线上数据库连接说明

## 1. 当前线上环境

- 项目后端运行在 Docker 容器中
- 相关容器：
  - `nginx-prod`
  - `php-prod`
  - `mysql-prod`
- MySQL 没有直接映射宿主机 `3306` 端口
- 因此不能直接用 Navicat 连接服务器公网 IP 的 `3306`

## 2. 为什么不能直接连

通过 `docker ps` 可以看到 `mysql-prod` 只暴露了容器内部端口：

```text
mysql-prod   mysql:8.0   ...   3306/tcp, 33060/tcp
```

这表示：

- MySQL 容器内部监听 `3306`
- 宿主机没有做 `3306:3306` 端口映射
- 外部客户端无法直接访问宿主机的 `3306`

所以线上数据库需要通过 SSH 端口转发访问。

## 3. 连接方式

### 3.1 第一步：确认 MySQL 容器 IP

在服务器执行：

```bash
docker inspect mysql-prod | grep IPAddress
```

示例结果里，MySQL 容器 IP 为：

```text
172.18.0.3
```

### 3.2 第二步：本机建立 SSH 端口转发

在本机终端执行：

```bash
ssh -L 3307:172.18.0.3:3306 root@223.6.251.191
```

含义：

- 本机开放一个临时端口 `3307`
- 通过 SSH 把本机 `3307` 转发到服务器上 MySQL 容器的 `172.18.0.3:3306`

注意：

- 这个终端窗口必须保持打开
- 关闭这个窗口后，Navicat 连接会断开

## 4. Navicat 配置

新建一个普通 MySQL 连接，不要使用 Navicat 自带的 SSH 隧道。

### 4.1 常规

- 连接名：`prod-db` 或其他自定义名称
- 主机：`127.0.0.1`
- 端口：`3307`
- 用户名：`navicat`
- 密码：`Navicat@123456`

### 4.2 SSH

- 不勾选 SSH 隧道
- 因为 SSH 已经由本机命令 `ssh -L ...` 接管

## 5. 数据库账号信息

### 5.1 root 初始信息

在服务器查看容器环境变量：

```bash
docker inspect mysql-prod | grep MYSQL
```

可看到：

```text
MYSQL_ROOT_PASSWORD=Root@123456
MYSQL_DATABASE=root
```

### 5.2 Navicat 专用账号

为了避免直接使用 root，建议新建一个专门用于数据库管理的账号。

在 MySQL 容器内执行：

```sql
CREATE USER 'navicat'@'%' IDENTIFIED BY 'Navicat@123456';
CREATE USER 'navicat'@'localhost' IDENTIFIED BY 'Navicat@123456';

GRANT ALL PRIVILEGES ON *.* TO 'navicat'@'%' WITH GRANT OPTION;
GRANT ALL PRIVILEGES ON *.* TO 'navicat'@'localhost' WITH GRANT OPTION;

FLUSH PRIVILEGES;
```

如果客户端对 MySQL 8 认证插件兼容性不好，可改为：

```sql
ALTER USER 'navicat'@'%' IDENTIFIED WITH mysql_native_password BY 'Navicat@123456';
ALTER USER 'navicat'@'localhost' IDENTIFIED WITH mysql_native_password BY 'Navicat@123456';
FLUSH PRIVILEGES;
```

## 6. 验证方法

### 6.1 在服务器内验证账号是否可用

```bash
docker exec -it mysql-prod mysql -unavicat -pNavicat@123456
```

如果这条命令能进入 MySQL，说明数据库账号本身没有问题。

### 6.2 Navicat 连接成功的正确姿势

必须满足以下条件：

- SSH 转发窗口保持开启
- Navicat 连接的是 `127.0.0.1:3307`
- Navicat 不再勾选 SSH 隧道
- 用户名密码使用数据库账号，不是服务器 SSH 密码

## 7. 常见错误排查

### 7.1 报错：`Access denied for 'password'`

说明 SSH 登录失败，检查：

- SSH 用户名
- SSH 密码或私钥
- 服务器 `22` 端口是否可访问

### 7.2 报错：`1045 Access denied for user ...`

说明已经到达 MySQL，但数据库认证失败，检查：

- 数据库用户名密码是否正确
- 是否误填成服务器 SSH 密码
- 当前连接目标是不是 `127.0.0.1:3307`
- Navicat 是否还勾着 SSH
- `navicat` 用户是否创建成功

### 7.3 本机执行 `mysql` 命令提示找不到

说明本机没安装 MySQL 命令行客户端，不影响使用 Navicat。

## 8. 最终可用方案

本项目当前稳定可用的连接线上数据库方式如下：

1. 在本机执行：

```bash
ssh -L 3307:172.18.0.3:3306 root@223.6.251.191
```

2. 在 Navicat 中新建普通 MySQL 连接：

```text
Host: 127.0.0.1
Port: 3307
User: navicat
Password: Navicat@123456
```

3. 不启用 Navicat 自带 SSH

## 9. 备注

- 如果 MySQL 容器 IP 变化，需要重新执行 `docker inspect mysql-prod | grep IPAddress`
- 如果后续把 MySQL 映射到宿主机端口，则可以调整为新的连接方式
- 当前推荐继续使用 SSH 转发，安全性更高
