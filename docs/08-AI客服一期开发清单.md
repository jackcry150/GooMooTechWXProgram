# AI客服一期开发清单

## 1. 一期目标

一期目标不是做完整客服平台，而是先把下面这条链路跑通：

- 小程序进入 AI 客服页
- 用户可发起售前 / 售后问题
- AI 可基于知识库回答高频问题
- 复杂问题可一键转人工
- 转人工继续复用现有企业微信客服能力
- 后台可维护知识库、查看会话记录、配置基础参数

一句话概括：

先做“AI分流 + 人工兜底 + 数据留痕”。

建议入参：

## 4.2 页面功能

### 4.2.1 初始展示

- 欢迎语
- 当前场景标签
- 快捷问题列表
- 历史消息列表

### 4.2.2 用户发送消息

- 输入文本后提交到 `/aiService/message/send`
- 前端先插入用户消息
- 返回成功后插入 AI 消息

### 4.2.3 转人工

- 点击“转人工”
- 调用 `/aiService/transfer/manual`
- 获取人工客服参数
- 小程序端调用现有 `wx.openCustomerServiceChat`

### 4.2.4 场景差异

售前：

- 显示商品相关快捷问题
- 可带 `productId`

售后：

- 若未登录，先登录
- 若带 `orderId`，自动显示“当前订单售后咨询”
- AI 查询订单状态、退款状态、物流状态时走后端鉴权

## 5.2 API 说明

### 5.2.1 创建会话

`POST /api/aiService/session/create`

请求参数：

```json
{
  "scene": "presale",
  "productId": 12,
  "orderId": 0,
  "sourcePage": "product"
}
```

返回示例：

```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "sessionId": 1,
    "sessionNo": "AICS202603180001",
    "welcomeMessage": "您好，我是智能客服，有什么可以帮您？"
  }
}
```

### 5.2.2 发送消息

`POST /api/aiService/message/send`

请求参数：

```json
{
  "sessionId": 1,
  "content": "这个商品什么时候发货？"
}
```

返回示例：

```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "reply": "当前商品为预售商品，具体发货时间请以商品详情页说明为准。",
    "needTransfer": 0,
    "confidence": 0.92,
    "hitKnowledgeId": 3
  }
}
```

### 5.2.3 会话详情

`GET /api/aiService/session/detail?id=1`

返回内容：

- 会话基本信息
- 消息列表
- 是否已转人工

### 5.2.4 快捷问题

`GET /api/aiService/faq/list?scene=presale`

返回内容：

- 当前场景可用快捷问题

### 5.2.5 转人工

`POST /api/aiService/transfer/manual`

请求参数：

```json
{
  "sessionId": 1,
  "reason": "user_request"
}
```

返回内容建议：

```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "corpId": "wwbc9e055c515dc9e5",
    "link": "https://work.weixin.qq.com/kfid/kfc8ed1ca5ca7bfa56c",
    "title": "售后人工客服"
  }
}
```

## 5.3 AI 回复逻辑一期建议

一期不要一开始就做复杂 Agent。

建议先用“规则 + 知识库 + 兜底”三段式：

### 第一步：识别场景与上下文

- 判断是售前还是售后
- 读取 `productId` / `orderId`
- 获取对应商品或订单基础信息

### 第二步：匹配知识库

- 按 `scene + keywords + status` 进行匹配
- 优先返回人工维护的标准答案

### 第三步：兜底策略

- 没匹配到则返回统一兜底话术
- 同时返回 `needTransfer = 1`

这样一期就算没接大模型，也能先上线“智能问答”第一版。

如果后续接大模型，则建议：

- 大模型只负责润色和组织答案
- 关键事实仍然来自本地知识库和订单数据

## 7.1 AI 客服配置表

表名建议：

- `mp_ai_service_config`

建表示例：

```sql
CREATE TABLE `mp_ai_service_config` (
  `id` int(11) NOT NULL AUTO_INCREMENT,
  `aiEnabled` tinyint(1) NOT NULL DEFAULT '1' COMMENT 'AI客服开关',
  `allowGuestPresale` tinyint(1) NOT NULL DEFAULT '1' COMMENT '游客是否允许售前咨询',
  `allowGuestAftersale` tinyint(1) NOT NULL DEFAULT '0' COMMENT '游客是否允许售后咨询',
  `welcomePresale` varchar(500) DEFAULT '' COMMENT '售前欢迎语',
  `welcomeAftersale` varchar(500) DEFAULT '' COMMENT '售后欢迎语',
  `fallbackReply` varchar(500) DEFAULT '' COMMENT '兜底回复',
  `transferButtonText` varchar(50) DEFAULT '转人工' COMMENT '转人工按钮文案',
  `lowConfidenceThreshold` decimal(5,2) NOT NULL DEFAULT '0.60' COMMENT '低置信度阈值',
  `unresolvedLimit` tinyint(2) NOT NULL DEFAULT '2' COMMENT '未解决次数阈值',
  `serviceTimeText` varchar(255) DEFAULT '' COMMENT '人工接待时间说明',
```

## 7.2 会话表

表名建议：

```sql
CREATE TABLE `mp_ai_service_session` (
  `id` int(11) NOT NULL AUTO_INCREMENT,

  `transferReason` varchar(100) DEFAULT '' COMMENT '转人工原因',
  `summary` varchar(1000) DEFAULT '' COMMENT '会话摘要',
  `lastMessageTime` datetime DEFAULT NULL COMMENT '最后消息时间',
```

## 9. AI 命中策略建议

一期先别做太复杂，建议按下面顺序：

### 9.1 精准规则

例如：

- 包含“退款”“退货”“发票”“尾款”“发货”“物流”等核心词
- 直接映射到固定意图

### 9.2 关键词匹配知识库

- 对用户问题做分词或简单关键词包含判断
- 命中知识库则返回标准答案

### 9.3 上下文补充

- 如果是售后且有订单，拼接订单状态说明
- 如果是售前且有商品，拼接商品信息说明

### 9.4 兜底

- 返回“这个问题需要人工客服进一步协助”
- 同时允许转人工

## 10. 权限与安全

## 10.1 登录要求

- 售前咨询：游客可用
- 售后咨询：必须登录
- 查询订单详情：必须校验 `userId == order.userId`

## 10.2 数据安全

- AI 不直接暴露完整订单敏感信息
- 仅返回必要字段
- 会话中不要记录支付敏感信息

## 10.3 风险控制

以下问题不要由 AI 直接最终裁决：

- 是否赔偿
- 是否特殊退款
- 是否补差价
- 是否质量责任归属

这些场景统一建议：

- 给出规则说明
- 引导转人工

- 前端新增一个独立页面 `pages/ai/customer.vue`
