**版本**：1.0

## 1. 环境变量（.env）

项目根目录为 `php/` 时，请在 `php/.env` 中配置下列变量（无则新建；**勿将 .env 提交到版本库**）。

### 1.1 数据库（必配）

| 变量名 | 说明 | 示例 |
|--------|------|------|
| DB_DRIVER | 数据库驱动 | mysql |
| DB_TYPE | 数据库类型 | mysql |
| DB_HOST | 数据库主机 | 127.0.0.1 |
| DB_NAME | 数据库名 | root |
| DB_USER | 用户名 | root |
| DB_PASS | 密码 | your_password |
| DB_PORT | 端口 | 3306 |
| DB_CHARSET | 字符集 | utf8mb4 |
| DB_PREFIX | 表前缀 | 空或 mp_（若表带前缀需与建表一致） |

说明：`config/database.php` 中通过 `env('DB_xxx', 默认值)` 读取，未设置时使用默认值（如 host 127.0.0.1、user root、空密码等）。

### 1.2 应用调试（建议生产关闭）

| 变量名 | 说明 | 示例 |
|--------|------|------|
| APP_DEBUG | 是否调试模式 | true / false |
| APP_TRACE | 是否显示 Trace（若启用） | true / false |

说明：生产环境建议 `APP_DEBUG=false`，避免暴露异常详情。

### 1.3 其他可能存在的变量

- 若使用 Redis：如 `CACHE_DRIVER=redis`、`REDIS_HOST`、`REDIS_PORT` 等，以实际 `config/cache.php` 为准。
- 若支付/微信等从环境读取：如 `WECHAT_APPID`、`WECHAT_SECRET`、汇付相关变量等，需参考项目内使用处或扩展文档配置。

## 2. 配置文件清单与说明

以下文件位于 `php/config/` 目录。

### 2.1 app.php 应用设置

- **作用**：应用命名空间、路由开关、默认应用、时区、异常模板、错误信息等。
- **常用项**：
  - `default_timezone`：默认 `Asia/Shanghai`。
  - `error_message`：非调试模式下的错误提示。
  - `show_error_msg`：是否向用户展示错误信息。
- **环境变量**：本文件内通常不直接读 env，如需可通过 `env('APP_DEBUG')` 在入口或其它处使用。

### 2.2 database.php 数据库

- **作用**：数据库连接配置。
- **环境变量**：见上表（DB_DRIVER、DB_TYPE、DB_HOST、DB_NAME、DB_USER、DB_PASS、DB_PORT、DB_CHARSET、DB_PREFIX）。
- **说明**：默认连接名为 `mysql`，可增加其它连接（如从库）。

### 2.3 cache.php 缓存

- **作用**：缓存驱动与连接。
- **默认**：`default => 'file'`，存储为文件。
- **stores**：如 `file` 的 path、prefix、expire 等；可增加 `redis` 等，并对应修改 `default`。

### 2.4 log.php 日志

- **作用**：日志通道、级别、格式等。
- **默认**：`default => 'file'`，写入文件。
- **path**：未写死时一般为 runtime 下 log 目录。

### 2.5 session.php 会话

- **作用**：Session 驱动（file/cache）、过期时间、前缀等。
- **说明**：管理后台登录态依赖 Session，部署时确保 session 可写（file 驱动时使用系统临时目录或指定 path）。

### 2.6 cookie.php Cookie

- **作用**：Cookie 有效期、域名、路径等。
- **说明**：与跨域、前后端分离时的 Cookie 行为相关时需关注 domain、secure 等。

### 2.7 view.php 模板

- **作用**：模板引擎类型（Think）、目录名、后缀、标签等。
- **说明**：后台 adm 使用 `view/adm/` 下模板，一般无需改。

### 2.8 filesystem.php 文件系统

- **作用**：本地/公共磁盘路径、URL。
- **说明**：上传若走 Think 文件系统，会用到此处；本项目上传多在 `public/uploads`，可与此处 public 盘对应。

### 2.9 route.php 路由

- **作用**：pathinfo 分隔符、是否强制路由、默认路由规则等。
- **说明**：未定义的路由按 ThinkPHP 默认 `应用/控制器/方法` 解析，如 `/api/order/list`。

### 2.10 middleware.php 中间件

- **作用**：全局与路由中间件注册。
- **说明**：若在 API 或 adm 中做了 CORS、鉴权等中间件，会在此或应用内注册。

### 2.11 lang.php 多语言

- **作用**：默认语言、多语言切换等。
- **说明**：未做多语言时保持默认即可。

### 2.12 trace.php 调试 Trace

- **作用**：调试栏展示、SQL 等（开发用）。
- **说明**：生产建议关闭或仅对管理员开放。

### 2.13 console.php 控制台

- **作用**：命令行相关配置（若有 think 命令）。
- **说明**：一般部署不依赖，按需保留。

## 3. 前端环境与配置

### 3.1 请求 BaseURL（uniapp）

- **文件**：`uniapp/utils/request.js`
- **说明**：`baseURL` 决定接口域名。
  - 生产：`https://api.goomooplay.com/api`
  - 开发：可改为 `http://本机或内网IP/api`（如 `http://127.0.0.1/api` 需在小程序开发设置中勾选“不校验合法域名”等）。

### 3.2 小程序 manifest / 环境区分

- 若使用 `process.env` 或 manifest 中不同环境变量，以项目内实际为准；通常通过修改 `request.js` 的 baseURL 或条件编译即可区分环境。

## 4. 配置检查清单（部署前）

- [ ] `php/.env` 已创建并配置 DB_*、APP_DEBUG。
- [ ] 数据库已创建，并执行 schema_optimized.sql。
- [ ] `php/runtime`、`php/public/uploads` 可写。
- [ ] 生产环境 APP_DEBUG=false。
- [ ] 小程序 request 合法域名已配置为后端域名（如 https://api.goomooplay.com）。
- [ ] 支付回调 URL 使用 HTTPS 且可从公网访问。
- [ ] 微信/汇付等密钥与 appid 已配置（在代码或 .env 中，按项目实际）。
